### 💀 CVE-2025-55182 (通称: React2Shell) のヤバさ

セキュリティ業界では「CVSS」というスコアで危険度を表すんだけど、これは**満点の 10.0 (Critical)** です。つまり、これ以上の危険度はありません。

|**項目**|**内容**|**ヤバさ**|
|---|---|---|
|**攻撃の難易度**|**低 (Low)**|誰でも簡単に攻撃できる。|
|**認証**|**不要 (None)**|ログインしてない部外者が攻撃可能。|
|**影響**|**完全掌握**|サーバー上で好きなコマンドを実行できる（RCE）。|
|**現状**|**攻撃観測中**|既に世界中でこの脆弱性を狙った攻撃が始まっている。|

#### 💥 何が起きるの？

React Server Components (RSC) がデータをやり取りする「Flightプロトコル」という仕組みに、**安全ではないデシリアライズ（データの復元処理）**のバグが見つかりました。

攻撃者は、特定の細工をしたHTTPリクエストを1回送るだけで、認証なしでサーバー内の任意のコードを実行できてしまいます。

つまり、**「サーバーの裏口が全開」**の状態です。

---

### 🎯 影響を受ける主なフレームワーク

React 19のエコシステムを採用しているものが対象です。特に以下の構成が危険です。

1. **Next.js** (App Routerを使用しているバージョン 15.x, 16.x など)
    
2. **Waku**
    
3. **React Router** (RSCを使用している場合)
    
4. その他、`react-server-dom-webpack` などのパッケージを使っているもの
    

### 🛡️ 今すぐやるべき対策

キミのNext.jsプロジェクトの `package.json` を確認して、依存関係をアップデートしてください！

Next.jsの場合:

最新のパッチバージョン（修正済みバージョン）へアップグレードが必要です。

Bash

```
# npmの場合
npm install next@latest react@latest react-dom@latest

# yarnの場合
yarn upgrade next react react-dom
```

※ 修正されたバージョン（例: Next.js 15.1.0以降など、各ベンダーの最新情報を確認）になっているか必ずチェックしてね！

### 💡 ひとこと

キミが作っている「Next.js + Rust」のチャットアプリ、もしNext.js側がサーバー（BFF）として機能しているなら、この脆弱性のド真ん中です！
